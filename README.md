# Speech Recognition with Added Noise

Распознавание русской речи с помощью Google Speech Recognition API и оценка влияния добавленного шума на качество распознавания.

## Описание проекта

Проект исследует, как добавление шума влияет на точность распознавания речи. Сравнивается качество распознавания чистых аудиозаписей и записей с искусственно добавленным шумом с помощью метрик **WER** (Word Error Rate) и **CER** (Character Error Rate).

## Принцип работы

1. **Генерация чистых аудиозаписей** — русская речь генерируется с помощью библиотеки `gTTS` (Google Text-to-Speech) и сохраняется в формате MP3.
2. **Добавление шума** — на чистые записи накладывается белый или розовый шум с заданным отношением сигнал/шум (SNR).
3. **Распознавание речи** — библиотека `SpeechRecognition` с Google Speech Recognition API распознаёт текст из аудиофайлов.
4. **Постобработка текста** — распознанный текст нормализуется: приведение к нижнему регистру, исправление частых ошибок, конвертация чисел в слова.
5. **Оценка точности** — вычисление метрик WER и CER для сравнения исходного и распознанного текста.

## Зависимости

Проект использует следующие Python-библиотеки:

| Библиотека | Назначение |

## Основные функции

### `add_noise(audio, snr_db, noise_type)`
Добавляет к аудиосигналу белый или розовый шум с заданным SNR (дБ).
- **audio** — numpy-массив (float32, нормализован [-1, 1])
- **snr_db** — отношение сигнал/шум в децибелах
- **noise_type** — тип шума: `"white"` (белый) или `"pink"` (розовый, спад 3 дБ/октаву)

### `post_process_text(text)`
Постобработка распознанного текста:
- Приведение к нижнему регистру и нормализация пробелов
- Капитализация первых букв предложений
- Исправление частых орфографических ошибок (`"щто"` → `"что"`, `"ево"` → `"его"` и т.д.)
- Конвертация чисел в слова (например, `"123"` → `"сто двадцать три"`)

### `recognize_from_file(path)`
Распознавание речи из аудиофайла:
- Конвертирует MP3 в WAV через `pydub`
- Использует Google Speech Recognition API (язык: `ru-RU`)
- Применяет постобработку текста

## Результаты

### Чистые записи
Библиотека `SpeechRecognition` отлично распознаёт чистые записи. Снижение метрик вызвано в основном отсутствием знаков препинания в исходных текстах.

### Записи с шумом
При добавлении шума качество распознавания значительно ухудшается. Однако числительные распознаются неожиданно хорошо даже при сильном зашумлении (SNR = -10 дБ), несмотря на плохую слышимость на слух.

## Метрики оценки

- **WER (Word Error Rate)** — доля ошибочно распознанных слов относительно общего количества слов в исходном тексте.
- **CER (Character Error Rate)** — доля ошибочно распознанных символов относительно общего количества символов в исходном тексте.

Обе метрики вычисляются с помощью библиотеки `jiwer`.

## Выводы

1. Белый шум не отражает реалистичных условий, однако позволяет оценить устойчивость распознавания к помехам.
2. Google Speech Recognition API хорошо справляется с чистой русской речью.
3. При сильном шуме точность падает, но отдельные элементы (цифры) распознаются на удивление хорошо.
4. Постобработка текста помогает улучшить финальный результат распознавания.

## Использование

Откройте ноутбук `Speech_recognition_with_added_noise.ipynb` и выполните ячейки последовательно.

Для экспериментов можно менять:
- `snr_db` — уровень шума (по умолчанию -10 дБ)
- `texts` — список текстов для генерации аудио
- `noise_type` — тип шума (`"white"` или `"pink"`)
